System Overview

ReviewGuard is a hybrid AI system that combines three independent detection layers to identify synthetic and fraudulent product reviews with 92.35% accuracy. The system is designed for e-commerce platforms that require high-confidence fraud detection with minimal false positives.

Unlike single-model approaches, ReviewGuard uses ensemble fusion to combine semantic analysis (BERT), behavioral graph analysis (GNN), and rule-based heuristics into a unified verdict system.

Three-Layer Architecture

Layer 1: BERT (Semantic Analysis)

Fine-tuned BERT-base model trained on 21,000 Amazon reviews. Analyzes linguistic patterns, sentiment velocity, and deceptive language markers. Standalone accuracy: 87.5%.

  • Detects unnatural sentiment patterns and emotional manipulation
  • Identifies generic praise vs. specific product feedback
  • Analyzes sentence structure and vocabulary diversity

Layer 2: GNN (Behavioral Graph Analysis)

Graph Neural Network trained on YelpChi fraud graph (40,432 nodes, 521,140 edges). Maps reviewer behavior patterns and coordinated campaigns. Standalone accuracy: 83.2%.

  • Detects coordinated review campaigns and bot networks
  • Analyzes temporal patterns and review velocity
  • Identifies suspicious reviewer behavior clusters

Layer 3: Heuristic Engine

Rule-based detection system with 6 linguistic signals. Provides interpretable boost to final score (capped at +50%).

  • Exclamation mark abuse (≥3 occurrences)
  • Vague superlatives (amazing, best, perfect, etc.)
  • Word repetition patterns
  • Duplicate sentences (copy-paste detection)
  • ALL CAPS overuse
  • Missing negative sentiment (no criticism)

Fusion Formula

final_score = (0.65 × BERT) + (0.35 × GNN) + heuristic_boost where: BERT ∈ [0, 1] (higher = more fake) GNN ∈ [0, 1] (higher = more fake) heuristic_boost ∈ [0, 0.5] (capped at +50%) final_score ∈ [0, 1] (threshold: 0.35)

The 65/35 weight distribution reflects BERT's higher standalone accuracy. Heuristic boost is additive and capped to prevent false positives.

Verdict States

FAKE

final_score > 0.35 and divergence < 0.35

GENUINE

final_score ≤ 0.35 and divergence < 0.35

UNVERIFIABLE

divergence ≥ 0.35 (models disagree)

UNVERIFIABLE is triggered when BERT and GNN strongly disagree (divergence ≥ 0.35). Rather than force a wrong verdict, the system flags for human review. This prevents false positives on edge cases.

API Reference

POST /predict

Analyze a single review and return verdict with confidence scores.

// Request { "review": "string (required)", "rating": number (optional, 1-5) } // Response { "verdict": "FAKE" | "GENUINE" | "UNVERIFIABLE", "confidence": number, "bert_score": number, "gnn_score": number, "heuristic_score": number, "divergence": number, "reasons": string[], "submitted_rating": number | null }
POST /scrape

Scrape and analyze reviews from Amazon or Flipkart product pages.

// Request { "url": "string (required)", "max_reviews": number (optional, default: 8, max: 15) } // Response { "platform": "amazon" | "flipkart", "total_reviews_analysed": number, "fake_count": number, "genuine_count": number, "fake_percentage": number, "trust_score": number, "verdict": "HIGH RISK" | "MODERATE RISK" | "LOW RISK", "reviews": array }
POST /bulk-analyze

Analyze multiple reviews from pasted text (separated by blank lines or ---).

// Request { "reviews_text": "string (required)" } // Response (same structure as /scrape)
