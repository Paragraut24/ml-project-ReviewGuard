# Contributing to ReviewGuard

Thank you for your interest in ReviewGuard!

## Important Notice

**ReviewGuard is proprietary software.** This repository is publicly visible for transparency and demonstration purposes, but the code is not open for contributions in the traditional sense.

## What You Can Do

### ✅ Allowed

- **Report Bugs** - If you find a bug, please open an issue
- **Request Features** - Suggest new features via GitHub issues
- **Ask Questions** - Use GitHub Discussions for questions
- **Share Feedback** - Let us know how we can improve
- **Security Reports** - Report security vulnerabilities (see SECURITY.md)

### ❌ Not Allowed

- **Pull Requests** - We do not accept code contributions
- **Forking for Deployment** - Deploying your own instance requires explicit permission
- **Code Reuse** - Using code from this project in other projects is prohibited
- **Model Extraction** - Attempting to extract or replicate our models is prohibited

## How to Report Bugs

When reporting bugs, please include:

1. **Description** - Clear description of the issue
2. **Steps to Reproduce** - Detailed steps to reproduce the bug
3. **Expected Behavior** - What you expected to happen
4. **Actual Behavior** - What actually happened
5. **Environment** - Browser, OS, device information
6. **Screenshots** - If applicable

**Example:**

```markdown
**Bug Description:**
The analysis fails when submitting a review with special characters.

**Steps to Reproduce:**
1. Go to /workbench
2. Paste a review containing emoji: "Great product! 😊"
3. Click "Execute Trace"
4. Error appears

**Expected Behavior:**
The review should be analyzed successfully.

**Actual Behavior:**
Error message: "Analysis failed. Please try again."

**Environment:**
- Browser: Chrome 120
- OS: Windows 11
- Device: Desktop
```

## How to Request Features

When requesting features, please include:

1. **Feature Description** - What feature you'd like to see
2. **Use Case** - Why this feature would be useful
3. **Proposed Solution** - How you envision it working (optional)
4. **Alternatives** - Other solutions you've considered (optional)

**Example:**

```markdown
**Feature Request:**
Add support for analyzing reviews in languages other than English.

**Use Case:**
Many e-commerce platforms operate globally and need to analyze reviews in multiple languages.

**Proposed Solution:**
Add a language selector dropdown that allows users to specify the review language before analysis.

**Alternatives:**
- Automatic language detection
- Multi-language support without user selection
```

## Code of Conduct

### Our Standards

- **Be Respectful** - Treat everyone with respect
- **Be Professional** - Keep discussions professional and on-topic
- **Be Constructive** - Provide constructive feedback
- **Be Patient** - Understand that responses may take time

- Harassment or discriminatory language
- Trolling or insulting comments

- Attempting to circumvent security measures
